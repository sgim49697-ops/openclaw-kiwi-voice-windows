# license-compliance.md - 라이선스 준수 정책

이 문서는 public 저장소 운영을 위한 최소 라이선스 점검 기준이다. 법률 자문이 아니라 작업자가 실수로 권한이 불명확한 자료를 커밋하지 않도록 하는 운영 규칙이다.

## 기본 원칙

- 출처와 라이선스가 확인되지 않은 외부 자료는 저장소에 넣지 않는다.
- 외부 문서나 README는 필요한 만큼만 요약하고 원문 링크를 남긴다.
- 코드 예제는 가능하면 직접 작성한다.
- 외부 코드를 그대로 가져와야 하면 원본 URL, 라이선스, copyright notice를 보존한다.
- 모델 가중치, 음성 데이터, 이미지, 아이콘, 폰트, 바이너리는 코드보다 라이선스 제한이 강한 경우가 많으므로 별도로 확인한다.

## 허용 가능성이 높은 자료

아래 자료도 반드시 notice와 조건을 확인한 뒤 사용한다.

- MIT
- Apache-2.0
- BSD-2-Clause
- BSD-3-Clause
- ISC
- CC0
- Public domain으로 명시된 자료

## 사용자 승인 없이 추가하지 않을 자료

- GPL, AGPL, LGPL 계열 코드
- CC-BY-SA, CC-BY-NC, CC-BY-ND 계열 자료
- 상용 EULA가 붙은 SDK, 모델, asset
- 라이선스가 표시되지 않은 GitHub gist, 블로그 코드, Stack Overflow 답변 원문
- 유료 폰트, 유료 이미지, 유료 음성 asset
- 웹사이트 문서 원문 대량 복사본
- 비밀값, API token, 쿠키, 세션 정보, 개인 인증 파일

## 현재 저장소 라이선스 상태

프로젝트 전체 라이선스는 아직 선택하지 않았다. `LICENSE` 파일이 추가되기 전까지는 공개 열람은 가능하지만, 재사용 권한을 부여한 것으로 해석하지 않는다.

향후 실제 코드와 배포 스크립트가 늘어나면 MIT 또는 Apache-2.0 같은 permissive license 중 하나를 명시적으로 선택한다.
